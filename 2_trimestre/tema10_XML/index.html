Resumen de XML

XML (Lenguaje de Marcado Extensible) es un estándar del W3C diseñado para almacenar, describir y compartir datos entre diferentes sistemas, separando la información de su presentación.

¿Qué es XML?

XML no es un lenguaje de programación ni una base de datos. Es un lenguaje de marcado de estructura libre, similar a HTML en apariencia, pero con un propósito diferente: HTML muestra datos y XML los transporta.

Características Principales

  • Estructura de árbol: Todo documento tiene una única raíz.
  • Etiquetas propias: No están predefinidas, el autor las inventa.
  • Texto plano: Es legible tanto por humanos como por máquinas.
  • Estricto: Debe estar bien formado para funcionar.

Estructura de un Documento

Un documento XML se divide en dos partes fundamentales:

  • Prólogo: Incluye la declaración XML (versión y codificación).
  • Cuerpo: Contiene el elemento raíz y todo el árbol de contenido.

Reglas de Sintaxis (Bien Formado)

Para que un XML sea procesado correctamente debe cumplir estas reglas:

  • Todas las etiquetas deben tener apertura y cierre.
  • Los atributos deben ir siempre entre comillas.
  • Distingue entre mayúsculas y minúsculas (Case Sensitive).
  • Los elementos deben estar correctamente anidados.

Validación (DTD y XSD)

Un documento está "validado" cuando, además de estar bien formado, cumple con unas reglas específicas de estructura y tipos de datos definidas en:

  • DTD: Definición de Tipo de Documento (más antiguo).
  • XSD: XML Schema (más potente y permite definir tipos de datos).

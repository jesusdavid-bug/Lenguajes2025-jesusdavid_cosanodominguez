Tema 12. Conversión y adaptación de documentos XML

XPath es un lenguaje estándar que sirve para evaluar expresiones, navegar y acceder a los nodos específicos dentro de documentos XML. Actúa como una ruta o camino para extraer datos de forma precisa.

1. Lenguajes de Transformación XSL

XSL (eXtended Stylesheet Language) es una ampliación orientada a XML. Sus principales componentes son:

  • XPath: Utilizado para evaluar expresiones y localizar elementos.
  • XSLT: Sirve para transformar el documento XML en HTML o en otro XML.
  • XSL-FO: Permite transformar los datos a otros formatos de salida, como por ejemplo PDF.

2. El Árbol del Documento XML

Los documentos XML se estructuran como un árbol genealógico compuesto por distintos tipos de nodos:

  • Nodo Raíz: Es el origen del documento y siempre se representa con /.
  • Nodos Elemento, Atributo y Texto: Representan las etiquetas, sus propiedades y el contenido de las mismas.
  • Relaciones: Los nodos pueden ser ascendientes (padres), descendientes (hijos) o hermanos.

3. Sintaxis XPath

Una expresión XPath es una cadena de texto estructurada jerárquicamente. Se divide en:

  • Eje: Selecciona qué elementos o atributos buscar (ej. /concesionario/coche).
  • Predicado: Se escribe entre corchetes [...] y aplica condiciones o filtros al eje (ej. [@matricula="8280KWT"]).
  • Selección de nodos: Indica el resultado final a mostrar, como text() o @atributo.

4. Formatos de Intercambio de Datos

Además de XML, existen otros formatos populares para manejar información:

  • JSON (.json): Formato ligero y legible basado en pares clave-valor y arrays, muy eficiente en aplicaciones web.
  • CSV (.csv): Formato tabular, simple y estructurado en texto plano, donde los valores se separan por comas u otros delimitadores.
  • YAML (.yaml / .yml): Formato que utiliza la indentación para organizar los datos jerárquicamente; es menos verboso y muy común en archivos de configuración.
  • XML: Formato jerárquico basado en etiquetas, muy flexible pero más verboso.
